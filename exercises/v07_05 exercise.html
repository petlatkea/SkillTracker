Exercise - Random position (and speed)

Expand the CoinClicker mini game with randomness

You are going to implement my Coinclicker-game, but in very small steps. In previous exercises you added animations, events, and counting points and lives.
For this version, you are only going to worry about what happens to the coins, bomb, and heart, when they reach the bottom of the screen: they should still start over from the top, but at a different horizontal position.

Download the zip-file: coinclicker_random.zip and unpack it to your exercises folder.

If you really want to, you can work with your existing files - you only need to edit the CSS: add the nine .position classes, and remember to remove any left: properties from all the containers.

The premise is simple: When any element reaches the bottom of the screen, i.e. when its' animation is about to restart, give it a random position, with one of the position-classes.
Don't forget to remove all the existing position-classes!

You are probably going to have to create a separate function for each container on the screen, but start out with one, and see if you can use a similar solution for the rest.

If you are feeling adventurous, also try to set random speeds on each element!

You can combine this exercise with CoinClicker game over !
