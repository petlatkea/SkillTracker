Exercise - CoinClicker game over

Expand the CoinClicker mini game with game over

You are going to implement my Coinclicker-game, but in very small steps. In previous exercises you added animations, events, and counting points and lives.
For this version, you are only going to worry about what happens when too many lives has been lost. When there are no more lives left to lose, the game should be over, and it shouldn't be possible to click anything else.
Also, the player shouldn't be able to click the heart and gain extra lives, if she already has all hearts full.

Download the zip-file: coinclicker_gameover.zip and unpack it to your exercises folder.

This folder includes a basic JavaScript-file that counts lives up and down, when either the heart or bomb is clicked. You can also use your own solution to the Coinclicker Score-exercise, with counting of lives and points.
There's no animation in this exercise, but feel free to combine it with other exercises, to make an almost complete CoinClicker mini game.

Make a function called gameOver(), and call that function when the player has lost too many lives. The function should disable all clicking, and display a simple "Game Over"-message in the console.
Also fix the clickHeart() function, so it doesn't add lives beyond the number of hearts.

Use if statements!

You can combine this exercise with CoinClicker random !
